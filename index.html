App Store

Lessons 8 – 10
08

iOS Wrapper — WebView App in Xcode

Introduction to Xcode and the basics of Swift. We wrap the existing web game in a WKWebView iOS app — no rewriting needed. The game runs exactly as-is inside a native iOS shell. We set up the project, configure the bundle identifier, and test on the simulator. We also walk through the Apple Developer account setup and what to expect from the review process.

Swift Xcode WKWebView iOS simulator
09

App Store Assets & Metadata

We create all required App Store assets — app icon set, launch screen, and screenshots for different device sizes. Vasco Connor writes the app name, description, keywords, age rating, and selects the right category. We walk through App Store Connect together and set up the app listing end to end.

App Store Connect App icons Screenshots Metadata

TestFlight, Review & Submission

We upload the build via TestFlight for internal testing and do a final review checklist — privacy policy basics, content guidelines, and age rating. Once everything is confirmed, we submit for App Store review. Vasco Connor leaves with a complete, submitted app and a clear understanding of how to manage future updates independently.

Terms Explained

A plain-English guide to the technical terms used in this program.

PWA
Progressive Web App
A website that behaves like a real app. Once set up, visitors can tap "Add to Home Screen" on their phone and the game appears like a normal app — with its own icon, no browser bar, and the ability to work offline. No App Store required for this step.
Service Worker
Background helper
A small invisible script that runs in the background of the game. It allows the game to load even when there's no internet connection, by saving a copy of the game on the device. Think of it as the game's personal offline backup.
Vercel
Hosting platform
The platform where Vasco Connor's game and landing page will be published and made available to the world. It gives the project a real web address (URL) that anyone can visit, and keeps the site running 24/7. The free plan is sufficient for this project.
Seedance
AI video tool
An AI tool made by ByteDance (the same company behind TikTok) that generates cinematic video clips from text descriptions or images. Vasco Connor will use it to create the promotional video for his game — describing scenes in words and letting the AI bring them to life in high quality.
Cursor
AI code editor
The tool Vasco Connor writes his code in. It's like Microsoft Word, but for programming — and it has a built-in AI assistant that helps suggest, fix, and explain code as Vasco Connor works. Vasco Connor already has a subscription to this.
Xcode
Apple's dev tool
Apple's official free software for building iPhone and iPad apps. Vasco Connor will use it to wrap his existing web game into an iOS app — essentially putting it in an Apple-approved container so it can be submitted to the App Store. Requires a Mac to run.
TestFlight
Apple's testing tool
Apple's official way to test an app before it goes live on the App Store. It lets Vasco Connor install the app on a real iPhone and check that everything works correctly before submitting it for Apple's review. Think of it as a private "soft launch."
Apple Developer Program
App Store membership
A paid annual membership ($99 USD/year) required by Apple before any app can be submitted to the App Store. It gives Vasco Connor the ability to publish apps under his own name and access the tools needed for submission. This is the only mandatory purchase for the App Store phase.
Landing Page
Promotional website
A single-page website designed to showcase and promote the game. It will include the promo video, a link to play the game, a description of the sustainability theme, and a call to action. This is the public face of the project — something Vasco Connor can share with anyone.
